### IFC-V1-001 Strict Criteria

- Ready startup returns a `HostStatusResponse` only after state directory access, SQLite migrations, settings/bind validation, tmux discovery, registry reconciliation, and output-reader startup for live sessions have passed.
- Failure startup throws a typed startup error with bounded API error details and a non-ready host status; it must not return a ready-looking status after invalid state dir, invalid bind/port/settings, migration failure, missing tmux, or output-reader startup failure.
- Restart reconciliation marks missing durable sessions stale, keeps stopped sessions stopped, ignores unmanaged HostDeck-looking tmux targets without importing them, and reports stale/unmanaged counts in startup evidence.
- Default successful startup remains local-first: localhost bind, LAN disabled unless stored settings say otherwise, storage/tmux/stream health all `ok`, and stale sessions visible instead of hidden.
- Validation must include automated ready-path and negative startup tests plus command evidence in `artifacts/ifc-v1-001-startup-readiness.md`.
